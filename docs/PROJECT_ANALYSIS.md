# Анализ проекта PT_LEP_unix

## 1. Текущее назначение

Проект реализует Ansible-роль для подготовки Linux-хостов к передаче событий безопасности в SIEM:
- контроль и конфигурация audit-подсистемы;
- доставка событий через syslog-стек;
- поддержка offline-инсталляции пакетов.

## 2. Сильные стороны

1. Практичный production-фокус: есть fallback на локальные пакеты.
2. Широкий покрываемый список ОС (Deb/RHEL/Ubuntu/Astra/Alt/REDOS).
3. Шаблоны маршрутизации событий на несколько SIEM-агентов.
4. Теги и разбиение task-файлов упрощают выборочный запуск.

## 3. Технические риски и ограничения

1. Логика роли содержит заметную долю shell-команд и distro-specific ветвлений, что повышает стоимость сопровождения.
2. В проекте пока нет встроенного CI-контура (`ansible-lint`, syntax-check, smoke-тесты).
3. В текущем окружении разработки отсутствует `ansible-playbook`, поэтому локальная syntax-валидация не выполняется автоматически.

## 4. Рекомендуемые улучшения

### Выполнено (2026-03-04)
1. Исправлен `packages_dirs` для REDOS/RED и синхронизирован с `files/packages`.
2. Добавлены post-install проверки в offline ветках (`package_facts`/`assert` и проверки бинарников).
3. Определение syslog-демона переведено с `pgrep` на проверку установленных сервисов (`service_facts`).
4. Строковые сравнения версий auditd заменены на безопасные сравнения (числовой major/version test).
5. Inventory санитизирован до безопасного шаблона без внутренних адресов.
6. Default SIEM endpoint в репозитории заменен на placeholder.
7. Закрыты мелкие quality-замечания (`task name typo`, shell-конвейеры в grep-checks).
8. Расширена матрица поддержки ОС: RedHat `7..9`, CentOS `7..10`, OracleLinux `7..9`, Debian `6..13`, REDOS/RED `7..9`.
9. Режим установки `with_repos` для RHEL-like переведен на package-agnostic подход.
10. В `without_repos` добавлена явная проверка наличия mapping и локальных пакетов для текущей версии ОС.
11. Добавлены офлайн-наборы пакетов для устаревших Debian (`6/7/8`) и включены соответствующие mapping записи.
12. Добавлены офлайн-наборы для актуального Debian `13`, CentOS `9/10` и совместимые REDOS `8/9` bundles; обновлены mapping записи для fully-offline сценариев.
13. Расширена поддержка AstraLinux (1.x-2.x) и добавлен offline compatibility mapping для релизов `1.5/1.6/1.7/1.8/2.12`.

### Приоритет P2 (актуальный)
1. Вынести повторяющуюся install-логику в reusable include/role helpers.
2. Перейти с raw shell-команд на idempotent модули там, где возможно.
3. Добавить `requirements.yml` для зависимостей (например `community.general`).

## 5. Общий вывод

Кодовая база решает практическую задачу и пригодна к эксплуатации, но нуждается в небольшом цикле стабилизации вокруг:
- операционной гигиены inventory,
- сокращения shell-heavy логики,
- автоматических проверок качества.
